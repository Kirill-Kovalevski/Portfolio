Short, practical ideas you can apply the same day.

  1. 3-step repro (reproduce → reduce → record).
  2. Micro red/green tests around the bug; commit each green.
  3. Binary search the blame with git bisect.
  4. Contracted logs: shape events as {evt,id,t,payload}.
  5. One hypothesis per change; write it above the diff.
  6. Layer proof: UI → API → DB, one command each.
  7. Five-minute duck, then a 10-minute reset if stuck.
  • Ship images as AVIF/WebP with srcset.
  • Inline 5–8KB of critical CSS; defer the rest.
  • preconnect to fonts/APIs; preload the main font.
  • Prefer native (dialog, URL, details).
  • content-visibility:auto below the fold.
  • Split by route; lazy non-critical chunks.
  • Cache immutables; add a simple service worker.
  • Track LCP/CLS/INP; fix outliers, not averages.
  • Stabilize props; memo hot paths.
  • < 100KB JS at first paint—fight for it.
  1. Show units in labels: “Limit (MB)”.
  2. State intent on buttons: “Create project” > “Submit”.
  3. Blame the system in errors: “We couldn’t save…”.
  4. Preview consequences: “Invites 3 people • sends email”.
  5. Use thresholds: “≈ 2–3 min” beats “~120 sec”.
  6. Put math in words: “Totals exclude archived items”.
  7. Progressive disclosure: show “Advanced” last.
  8. Reassure near risky actions: “Undo for 30s”.
  9. Good defaults win: choose for the user when safe.
  • Write an OpenAPI/JSON-Schema before code; generate stubs.
  • Add compat tests: yesterday’s contract must pass today.
  • Version by capability flags over URLs when possible.
  • Log schema drift in CI (fail on breaking changes).
  • Ship typed clients; devs copy/paste less, break less.

Schedule refactors like workouts. Every Tuesday:

  1. Pick a 30-minute slice (one file / module).
  2. Write a safety net test first.
  3. Refactor only what the test touches.
  4. Open a tiny PR; keep a running “debt ledger”.
  5. After 8 Tuesdays, review the ledger and celebrate deltas.

Spend one focused hour per weekday making a durable note:

  • One concept, one diagram, one code snippet.
  • Link it to two other notes (“backlinks”).
  • At month’s end, publish the best 3 as blog posts or gists.
  • Result: a compounding personal knowledge base & portfolio.
  • Daily system: 3 outcomes → 25-minute sprints → 5-minute log.
  • Weekly system: demo something every Friday (even ugly).
  • Hiring system: one new contact/day, one PR/week, one article/month.
  • Health system: alarm to stop coding at 18:30; walk while thinking.
  • Systems reduce motivation debt—momentum replaces willpower.
